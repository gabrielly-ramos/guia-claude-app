Configurando o Cowork para começar a trabalhar

Esse é o padrão que o time de TI está adotando para todo mundo usar o Cowork do mesmo jeito — sem depender de como cada um "descobriu sozinho". Se você é iniciante, comece exatamente por aqui.

1 Antes de começar: organize a pasta do projeto

O Cowork trabalha dentro de uma pasta do seu computador — ele lê, cria e organiza arquivos ali. Por isso, a primeira regra é: nunca comece a trabalhar "solto", sempre dentro de uma pasta de projeto.

Um projeto contínuo = uma pasta própria

Se você vai voltar nesse assunto mais de uma vez (um dashboard, uma documentação, uma consultoria), crie uma subpasta só para ele. Isso mantém o contexto e os arquivos gerados sempre no mesmo lugar.

Pergunta pontual não precisa de pasta

Se é uma dúvida rápida ou algo que não vai gerar arquivo nenhum (ex: "o que preciso revisar hoje?"), pode ficar solto — não crie pasta para isso.

Não encaixou em nada existente? Crie uma pasta nova

Nunca jogue um arquivo solto na raiz só porque não sabe onde encaixar. Uma subpasta nova e bem nomeada é sempre melhor que bagunça.

Exemplo real usado aqui na empresa

meu-workspace/
├── Dev-SQL/              → projetos técnicos (cada um em sua subpasta)
├── Documentacao/         → documentação geral, fora de projeto técnico
├── Reunioes-Briefings/   → pautas e atas
└── Estudos/              → material de estudo pessoal

2 Nomeie a sessão certo

Ao abrir uma sessão nova no Cowork, dê um nome claro. Isso é o que permite encontrar a conversa certa depois, em vez de vasculhar uma lista de "Nova conversa", "Nova conversa (2)"...

[Área] Assunto curto
  • [Sankhya] Discovery inicial
  • [TCRPRO] Documentação AD
  • [Reunioes] Pauta GP Empresarial

3 Recursos essenciais que você vai usar sempre

Não precisa decorar sintaxe — no Cowork você pede as coisas em português normal. Aqui estão as frases/comandos que resolvem 90% do dia a dia.

"crie um artifact com..."

Gera uma página que fica salva e pode ser reaberta e atualizada depois — use para dashboards, status de projeto, checklists. Diferente de uma resposta de chat comum, que "expira".

"lembre disso"

Faz o Cowork guardar uma preferência ou informação para as próximas conversas — não precisa repetir contexto toda vez.

"agende isso para toda [dia/hora]"

Cria uma tarefa recorrente (ex: briefing toda segunda 9h) em vez de você pedir manualmente todo dia.

"/" (barra) no início da mensagem

Abre a lista de atalhos/comandos disponíveis na sua instalação — vale sempre dar uma olhada no que existe.

"resuma o que fizemos até aqui"

Bom hábito antes de encerrar uma sessão longa — confirma que você e o Cowork estão alinhados sobre o que ficou pronto.

"explique antes de fazer"

Para ações mais arriscadas (apagar, sobrescrever, enviar), peça explicação antes — o Cowork já pede confirmação nesses casos, mas você pode reforçar.
