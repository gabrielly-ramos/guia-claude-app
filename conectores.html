Usando conectores (MCP) para trabalhar com dados reais

Um conector é o que permite ao Claude conversar diretamente com um sistema da empresa — sem você precisar exportar planilha, copiar tela ou descrever o dado manualmente. Aqui usamos o Jira como exemplo, porque é um conector de uso geral, sem restrição de acesso. Alguns conectores da empresa são restritos a times específicos (ex: sistemas internos que só o time de dev acessa) — confira com seu time de TI antes de assumir que um conector está liberado para você.

1 O que muda quando existe um conector

Sem conector, o Claude só sabe o que você digitar ou colar na conversa. Com o conector do Jira ativo, ele consegue consultar (e em alguns casos alterar) os projetos e tarefas de verdade — sem você precisar abrir o Jira e copiar cada informação manualmente.

Cada conector expõe um conjunto fixo de ferramentas

Não é acesso livre ao sistema — é um contrato específico. O conector do Jira, por exemplo, expõe: listar projetos, buscar tarefas com uma linguagem de consulta própria (JQL), ver o detalhe de uma tarefa, e também criar, comentar ou mover tarefas.

Aqui a escrita existe — e por isso pede confirmação

Diferente de um conector de banco de dados (que costuma ser só leitura), o Jira permite criar tarefa, comentar e mudar status. São ações reais e visíveis para o time — o Claude sempre mostra o que vai fazer antes de confirmar.

Ver quais conectores existem: /mcp

Dentro de uma sessão do Cowork ou do Code, o comando /mcp lista os conectores configurados e se estão autenticados. Se um conector aparecer sem autorização, alguém do time responsável precisa autorizar antes de usar — e alguns conectores permanecerão fora do seu acesso por serem restritos a outro time.

2 As ferramentas do conector Jira

Não é preciso saber os nomes técnicos — basta pedir em português o que você quer. Mas entender o que cada ferramenta faz ajuda a formular o pedido certo.

listar projetos

Mostra os projetos do Jira que você tem acesso para ver — nome, chave (ex: PROJ) e os tipos de tarefa que cada um usa (Bug, História, Épico...).

buscar tarefas com JQL

JQL é a "linguagem de busca" do Jira — algo como project = PROJ AND status != Done. Você não precisa escrever isso: basta descrever o que quer ("bugs abertos de prioridade alta no projeto X") e o Claude monta a consulta.

ver detalhes de uma tarefa

Traz a descrição completa, comentários, responsável e histórico de uma tarefa específica a partir da chave (ex: PROJ-123).

criar / comentar / mover tarefa

As únicas ações que escrevem no Jira. Sempre aparecem para sua confirmação antes de serem executadas — revise responsável, projeto e texto antes de aprovar.

3 O fluxo certo: da pergunta ao painel

Você não precisa saber JQL de cor. O caminho confiável é descrever o que quer em português e deixar o Claude descobrir o projeto certo e montar a consulta:

Exemplo de conversa

Você: "quero um painel com os itens de prioridade alta em aberto do projeto X"

1. Claude lista os projetos → encontra "Projeto X" (chave PROJ)
2. Claude monta a busca: project = PROJ AND statusCategory != Done
   AND priority in (Highest, High)
3. Claude traz os resultados: status, tipo, responsável, resumo
4. Você pede para agrupar por status e por responsável
5. Você pede para virar um artifact

Peça para ele explicar antes de agir

Para qualquer ação que escreve (criar tarefa, comentar, mover status), peça "me mostre o que vai ser criado/alterado antes de confirmar" — o Claude já pausa para isso, mas vale reforçar em ações importantes.

JQL tem operadores próprios

Termos como statusCategory, priority in (...) e ORDER BY fazem parte da sintaxe do Jira — você não precisa memorizar, mas ajuda saber que existe essa camada quando for revisar uma consulta.

Dado sensível pede cuidado ao publicar

Nome de responsável, prioridade e prazos internos de tarefas são informação de time. Se o resultado for virar um artifact ou relatório compartilhado, confirme com quem pode vê-lo antes de mandar para fora do time.

4 Projeto prático — do conector ao artifact (15 min)

A melhor forma de fixar é repetir o fluxo completo uma vez, do zero, numa pergunta sua. Marque cada passo conforme for concluindo.
