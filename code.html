Configurando o Claude Code para começar a programar

Esse é o padrão do time de TI para quem vai usar o Claude Code no dia a dia de desenvolvimento. Se você já programa mas está travando na transição para gerar código com IA, comece exatamente por aqui.

1 Antes de começar: prepare o projeto

O Claude Code roda no terminal, dentro da pasta do seu projeto. Ele lê o código, edita arquivos, roda comandos e mexe no git — sempre dentro daquela pasta. Por isso a preparação importa mais aqui do que no Chat ou no Cowork.

Sempre dentro de um repositório git

O Claude Code funciona melhor com histórico de versão disponível — ele consegue ver diffs, criar commits e você consegue desfazer qualquer coisa que não gostar.

Primeira coisa a rodar: /init

Gera um arquivo CLAUDE.md na raiz do projeto, documentando automaticamente a estrutura, convenções e comandos do seu código. Ele lê esse arquivo em toda sessão futura.

Leia o diff antes de aceitar

Toda edição de código aparece para revisão antes de ser aplicada. Não aceite no automático — é assim que você aprende o que ele está fazendo e pega erros cedo.

Instalar e abrir pela primeira vez

npm install -g @anthropic-ai/claude-code

cd caminho/do/seu-projeto
claude

# dentro da sessão, primeiro comando:
/init

2 Entenda como ele pede permissão

Diferente de gerar código num chat comum e colar você mesmo, o Claude Code executa ações de verdade no seu computador. Por segurança, ele para e pergunta antes de qualquer coisa arriscada — isso não é lentidão, é o que te dá controle.

Ações simples (ler arquivo, rodar teste)

Costumam ser autorizadas direto, sem te interromper.

Ações que mudam algo (editar, criar arquivo)

Mostram o que vai mudar e pedem seu "sim" antes de aplicar.

Ações arriscadas (git push, deletar, force)

Sempre pedem confirmação explícita — e é esperado que você leia com atenção antes de aprovar.

3 Recursos essenciais que você vai usar sempre

Também dá para pedir a maioria das coisas em português normal — mas esses comandos e frases resolvem o dia a dia mais rápido.

/init

Gera o CLAUDE.md documentando seu projeto. Rode uma vez em cada repositório novo.

"não edite ainda, só me explique seu plano"

Força uma etapa de planejamento antes de mexer no código — essencial em tarefas maiores ou quando você quer entender a abordagem antes de aprovar.

/clear

Limpa o contexto da conversa ao trocar de assunto — evita que ele misture uma tarefa antiga com a nova.

"rode os testes antes de finalizar"

Pede verificação real (testes, build, lint) em vez de considerar a tarefa pronta só porque o código foi escrito.

"crie um commit dessa mudança"

Ele monta a mensagem e mostra antes de confirmar — revise a mensagem e os arquivos incluídos.

/help

Lista todos os comandos disponíveis na sua instalação — varia um pouco conforme plugins do time.

4 Projeto prático — aprenda fazendo (20 min)

Use um projeto pequeno que você já conhece bem — assim você consegue avaliar se as respostas fazem sentido, em vez de confiar às cegas. Marque cada passo conforme for concluindo.
