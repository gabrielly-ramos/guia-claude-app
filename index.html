Getting Started

Como usar o Claude no dia a dia

Existem três formas principais de trabalhar com o Claude. Cada uma serve para um tipo de tarefa diferente — entender essa diferença é o primeiro passo para usar bem.

Chat

Uma conversa, sem compromisso com arquivos ou projeto.

É como mandar mensagem para um colega que sabe de tudo um pouco. Você pergunta, pede uma opinião, um resumo, uma ideia — e recebe a resposta ali mesmo, sem nada sendo alterado no seu computador.

Sessão de conversa isolada, sem acesso a arquivos, terminal ou ferramentas do sistema. O contexto vive só naquela conversa. Ideal para consultas pontuais, revisão de trechos colados, brainstorm e explicações de conceitos.

Quando usar

  • Tirar uma dúvida rápida
  • Pedir uma segunda opinião sobre uma decisão
  • Revisar ou reescrever um texto
  • Aprender um conceito novo

Cowork

Um assistente que trabalha dentro das suas pastas de verdade.

Diferente do chat, o Cowork "mora" dentro de uma pasta do seu computador. Ele lê, organiza e cria arquivos — documentos, planilhas, anotações de reunião — e lembra do contexto do projeto entre uma sessão e outra.

Sessão com acesso ao sistema de arquivos e a integrações (MCP), memória persistente por pasta/projeto e histórico entre sessões. Focado em produtividade geral — não pressupõe que a tarefa envolva escrever código.

Quando usar

  • Organizar documentação de um projeto
  • Manter contexto entre várias conversas sobre o mesmo assunto
  • Gerar relatórios, atas e planilhas a partir de dados reais
  • Tarefas que não são código, mas precisam de arquivos

Code

Quem de fato escreve, roda e testa o código.

É a versão voltada para desenvolvimento de software. Ele escreve código, executa comandos, roda testes e mexe no controle de versão (git) — sempre te mostrando o que está fazendo antes de qualquer coisa mais arriscada.

CLI com acesso a shell, edição de arquivos, execução de build/testes e integração git. Suporta subagentes, hooks e ferramentas MCP. Pensado para o ciclo real de desenvolvimento: ler código existente, editar, rodar, verificar.

Quando usar

  • Implementar ou corrigir uma funcionalidade
  • Rodar testes e verificar se algo quebrou
  • Explorar um código que você não escreveu
  • Automatizar tarefas de terminal/git

Não sabe qual escolher?

  • Só uma pergunta ou ideia, sem precisar de arquivos? Use o Chat.
  • Precisa organizar arquivos ou manter contexto de um projeto não-técnico? Use o Cowork.
  • Precisa escrever, rodar ou testar código de verdade? Use o Code.
